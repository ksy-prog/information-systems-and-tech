# Основные принципы построения компьютерных сетей и их архитектуры

---

## Введение

Компьютерная сеть — это совокупность компьютеров и других устройств, соединённых между собой каналами связи для обмена данными и совместного использования ресурсов. В современном мире компьютерные сети являются основой цифровой коммуникации, от локальных сетей внутри офиса до глобальной сети Интернет.

Изучение принципов построения компьютерных сетей включает понимание:
- физического размещения устройств (топологии)
- уровней взаимодействия (модели OSI и TCP/IP)
- архитектурных решений
- используемых протоколов

---

## Определения и основные понятия

### Сетевые абоненты и ресурсы

**Сетевые абоненты** — это конечные устройства в сети:
- Персональные компьютеры
- Серверы
- Мобильные устройства
- Периферийные устройства (принтеры, сканеры)

**Сетевые ресурсы** могут быть:
- **Аппаратные** (принтеры, диски, модемы)
- **Программные** (файлы, базы данных)
- **Сервисы** (электронная почта, веб-сервисы, видеоконференции)

### Модели построения сетей

#### 1. Одноранговые сети (Peer-to-Peer, P2P)

В одноранговых сетях все компьютеры равноправны. Каждый компьютер может быть одновременно источником и потребителем ресурсов.

**Типы**: 
1. Неструктурированные — участники связываются между собой случайным образом.
2. Структурированные — имеют чёткую архитектуру взаимосвязей между узлами, которая позволяет более эффективно осуществлять обмен данными.
3. Гибридные — объединяют отдельные элементы предыдущих двух видов, что позволяет обеспечить высокий уровень производительности, устойчивости и децентрализации. 

**Преимущества**:
- Простая реализация
- Низкие затраты на оборудование
- Отсутствие единой точки отказа

**Недостатки**:
- Сложное управление
- Низкая производительность при большом количестве компьютеров
- Слабая централизованная безопасность

#### 2. Архитектура клиент-сервер

В этой модели выделенные компьютеры выступают в роли серверов (поставщиков ресурсов), а остальные — в роли клиентов (потребителей ресурсов).

**Пример**: HTTP/HTTPS для веб-приложений, TCP/IP для общей сетевой коммуникации.

**Преимущества**:
- Централизованное управление
- Легче обеспечить безопасность
- Масштабируемость
- Эффективное использование ресурсов

**Недостатки**:
- Высокие затраты на серверное оборудование
- Зависимость от надёжности сервера
- Уязвимость к сбоям сервера

## Топологии компьютерных сетей

Топология сети определяет физическое и логическое расположение устройств и способ их соединения. Существует различие между **физической топологией** (реальное расположение кабелей) и **логической топологией** (маршруты передачи сигналов).

### Физическая топология звезда (Star)


                +----------------------+
                |   Коммутатор / HUB   |
                +----------+-----------+
                           |
          -----------------+-----------------
          |               |                |
   +------+------+  +-----+------+  +------+------+
   |   ПК 1       |  |   ПК 2     |  |    ПК 3    |
   +--------------+  +------------+  +------------+


**Характеристики**:
- Все устройства подключены к центральному узлу
- Центр — коммутатор, маршрутизатор или сервер
- Двусторонняя связь точка-точка

**Достоинства**:
- Отказ одного компьютера не влияет на работу сети
- Высокая производительность
- Легко добавлять новые компьютеры
- Простое обнаружение проблем

**Недостатки**:
- Выход из строя центра приводит к отказу всей сети
- Требуется больше кабеля
- Зависимость от производительности центрального узла

**Пример**: Локальная сеть в офисе с коммутатором в центре.

### Физическая топология кольцо (Ring)


        +--------+           +--------+
        |  ПК 1  |  ----->   |  ПК 2  |
        +--------+           +--------+
            ^                   |
            |                   v
        +--------+  <-----  +--------+
        |  ПК 4  |          |  ПК 3  |
        +--------+          +--------+


**Характеристики**:
- Устройства соединены в замкнутый цикл
- Сигналы передаются в одном направлении
- Каждый компьютер усиливает сигнал и передаёт дальше

**Достоинства**:
- Равномерное распределение нагрузки
- Хорошие показатели пропускной способности
- Не требуется центральный узел

**Недостатки**:
- Выход из строя одного компьютера разрывает кольцо
- Сложнее добавлять и удалять компьютеры
- Сложное диагностирование неисправностей
- Задержки распространения сигналов



### Физическая топология шина (Bus)

  Терминатор                             Терминатор
      |                                       |
      v                                       v
  ---------------------------------------------------  ← Общая шина (кабель)
      |           |               |              |
      |           |               |              |
  +---+---+   +---+---+       +---+---+      +---+---+
  | ПК 1 |   | ПК 2 |       | ПК 3 |      | ПК 4 |
  +-------+   +-------+       +-------+      +-------+



**Характеристики**:
- Все устройства подключены к одному кабелю (магистрали)
- Сигналы передаются всем узлам одновременно
- Требуются терминаторы на концах кабеля

**Достоинства**:
- Простая реализация
- Низкие затраты на кабель
- Отказ одного компьютера не влияет на сеть

**Недостатки**:
- Разрыв кабеля отключает всю сеть
- Низкая производительность при большом числе узлов
- Сложнее диагностировать проблемы
- Конфликты при одновременной передаче


### Ячеистая (полносвязная) топология (Mesh)

          +--------+
          |  ПК 1  |
          +---+----+
           /  |   \
          /   |    \
         /    |     \
   +----+-----+   +--+-----+
   |  ПК 2    |   |  ПК 3  |
   +----+-----+   +----+---+
         \    |        /
          \   |       /
           \  |      /
          +---+-----+
          |  ПК 4   |
          +---------+


**Характеристики**:
- Каждое устройство соединено со всеми остальными
- Может быть полная (all-to-all) или частичная ячейка

**Достоинства**:
- Высокая надёжность (много резервных путей)
- Низкие задержки
- Хорошая масштабируемость для отказоустойчивости

**Недостатки**:
- Очень затратна (много кабеля и портов)
- Сложное управление
- Сложная установка

**Пример**: Критичные сети, требующие максимальной надёжности.

### Древовидная (иерархическая) топология (Tree)

                      +---------------------+
                      |  Корневой коммутатор|
                      |    (уровень ядра)   |
                      +----------+----------+
                                 |
                   --------------+--------------
                   |                           |
          +--------+---------+        +--------+---------+
          |  Коммутатор 1    |        |  Коммутатор 2    |
          | (уровень доступа)|        | (уровень доступа)|
          +----+------+------+-       +----+------+------+
               |      |                   |      |
           +---+--+ +---+--+          +---+--+ +---+--+
           | ПК 1 | | ПК 2 |          | ПК 3 | | ПК 4 |
           +------+ +------+          +------+ +------+


**Характеристики**:
- Комбинация звезды и шины
- Иерархическое расположение узлов
- Один основной узел (корень)

**Достоинства**:
- Масштабируемость
- Поддержка приоритетов
- Относительно простое управление

**Недостатки**:
- Зависимость от узла более высокого уровня
- Усложненная топология по сравнению с простыми топологиями

**Пример**: Сеть из нескольких офисов, соединённых иерархически.

### Смешанная топология (Hybrid)

На практике часто используется комбинация различных топологий. Например:
- Физическая топология звезда, а логическая топология общей шины (классический Ethernet)
- Комбинация звезды и дерева для расширения сети

                        +---------------------+
                        |   Центральный узел  |
                        |   (Дата-центр)      |
                        +----------+----------+
                                   |
                 ------------------+------------------
                 |                                   |
        +--------+---------+               +--------+---------+
        | Коммутатор офиса A|             | Коммутатор офиса B|
        +----+------+-------+             +----+------+-------+
             |      |                          |      |
         +---+--+ +---+--+                 +---+--+ +---+--+
         | ПК1 | | ПК2 |                 | ПК3 | | ПК4 |
         +------+ +------+                 +------+ +------+

## Модель OSI

Модель **Open Systems Interconnection (OSI)** была разработана Международной организацией по стандартизации (ISO) для стандартизации сетевого взаимодействия. Модель описывает передачу данных в сетях через **7 уровней**.

### Принцип работы модели OSI

Основной принцип построения модели: на каждом уровне добавляется служебная информация (заголовок и хвостовик). Этот процесс называется **инкапсуляция**.

```
Отправитель (сверху вниз):
Данные [7 уровень]
└─> Данные + заголовок 7 уровня [6 уровень]
└─> Данные + заголовки 7-6 уровней [5 уровень]
└─> Данные + заголовки 7-5 уровней [4 уровень]
└─> ... и так далее до физического уровня

Получатель (снизу вверх):
Биты [1 уровень]
└─> Кадр + проверка ошибок [2 уровень]
└─> Пакет [3 уровень]
└─> Сегмент [4 уровень]
└─> ... и так далее до прикладного уровня
```

### Уровни модели OSI

#### 1. Физический уровень (Physical Layer)

**Функции**:
- Передача физических сигналов по сети
- Определение электрических и механических характеристик передачи

**Работает с**:
- Электрические сигналы в кабелях (напряжение, ток)
- Световые импульсы в волоконных кабелях
- Радиоволны в беспроводных сетях

**Примеры**:
- Кабели Ethernet (UTP, STP, коаксиальный)
- Коннекторы RJ45
- Повторители (repeaters)
- Концентраторы (hubs)

**Пример из жизни**: Когда вы подключаете кабель Ethernet к сетевой карте, вы работаете на физическом уровне.

#### 2. Канальный уровень (Data Link Layer)

**Функции**:
- Доставка данных на соседний узел в локальной сети
- Управление доступом к среде передачи (MAC — Media Access Control)
- Обнаружение и исправление ошибок
- Упаковка данных в кадры (frames)

**Работает с**:
- MAC-адреса (48-битные адреса типа 00:1A:2B:3C:4D:5E)
- Кадры с заголовком и контрольной суммой

**Примеры протоколов**:
- Ethernet
- Wi-Fi (IEEE 802.11)
- PPP (Point-to-Point Protocol)

**Пример из жизни**: Когда компьютер отправляет данные на принтер в локальной сети, используется канальный уровень для доставки данных через коммутатор.

#### 3. Сетевой уровень (Network Layer)

**Функции**:
- Доставка пакетов между узлами в разных сетях
- Маршрутизация (выбор пути передачи)
- Логическая адресация (IP-адреса)
- Фрагментация и сборка пакетов

**Работает с**:
- IP-адреса (например, 192.168.1.1)
- Маршруты между сетями
- Таблицы маршрутизации

**Примеры протоколов**:
- IP (IPv4, IPv6)
- ICMP (используется в ping)
- IGMP (для многоадресной рассылки)

**Пример из жизни**: Когда вы заходите на сайт, ваш компьютер использует сетевой уровень для доставки запроса на сервер в интернете через маршрутизаторы.

#### 4. Транспортный уровень (Transport Layer)

**Функции**:
- Надёжная или ненадёжная передача данных
- Управление потоком данных
- Контроль ошибок
- Мультиплексирование (отправка нескольких потоков по одному каналу)

**Работает с**:
- Порты (номера 0-65535, например 80 для HTTP, 443 для HTTPS)
- Сегменты данных
- Номера последовательности и подтверждение доставки

**Примеры протоколов**:
- TCP (надёжный протокол с подтверждением)
- UDP (быстрый ненадёжный протокол)

**Пример из жизни**: 
- TCP используется при отправке электронной почты (SMTP, POP3)
- UDP используется в онлайн-играх и потоковом видео для низкой задержки

#### 5. Сеансовый уровень (Session Layer)

**Функции**:
- Установка и прекращение сеансов (sessions)
- Синхронизация диалога между приложениями
- Управление обменом сообщениями

**Работает с**:
- Сеансы (соединения приложений)
- Точки синхронизации для восстановления после сбоя

**Примеры протоколов**:
- NetBIOS (NetBIOS Session Service)
- PPTP (для VPN)
- RPC (Remote Procedure Call)

**Пример из жизни**: Когда вы открываете удалённый рабочий стол через RDP, сеансовый уровень управляет этим соединением.

#### 6. Уровень представления (Presentation Layer)

**Функции**:
- Преобразование формата данных
- Шифрование и дешифрование данных
- Сжатие и распаковка данных
- Кодирование символов

**Работает с**:
- Форматы файлов (JPEG, MPEG, GIF)
- Шифрование (SSL/TLS)
- Компрессия данных

**Примеры**:
- SSL/TLS (для защиты веб-сайтов)
- JPEG (формат изображений)
- ASCII и Unicode (кодировки)

**Пример из жизни**: Когда вы видите защищённый сайт (https://), используется шифрование на уровне представления.

#### 7. Прикладной уровень (Application Layer)

**Функции**:
- Взаимодействие с приложениями
- Предоставление сетевых сервисов пользователям
- Интерфейс между пользователем и сетью

**Работает с**:
- Запросы и ответы приложений
- Данные в формате, понятном приложению

**Примеры протоколов**:
- HTTP (веб-браузеры)
- SMTP, POP3, IMAP (электронная почта)
- FTP (передача файлов)
- DNS (преобразование имён в IP-адреса)
- Telnet (удалённый доступ)

**Пример из жизни**: Когда вы отправляете электронное письмо, ваше приложение (например, Outlook) использует SMTP на прикладном уровне.

### Таблица уровней OSI

| Уровень | Название | Функция | Примеры |
|---------|----------|---------|---------|
| 7 | Прикладной | Сервисы приложений | HTTP, SMTP, FTP, DNS |
| 6 | Представления | Преобразование формата | SSL/TLS, JPEG |
| 5 | Сеансовый | Управление сеансом | NetBIOS, RPC |
| 4 | Транспортный | Надёжная передача | TCP, UDP |
| 3 | Сетевой | Маршрутизация | IP, ICMP |
| 2 | Канальный | Локальная доставка | Ethernet, Wi-Fi |
| 1 | Физический | Передача сигналов | Кабели, сигналы |

## Модель TCP/IP

Модель **TCP/IP** (также называется моделью ARPANET) — это практическая модель, на которой построен интернет. В отличие от концептуальной модели OSI, TCP/IP используется реально и состоит из **4 уровней**.

### Уровни модели TCP/IP

#### 1. Уровень сетевого интерфейса (Network Interface/Link Layer)

Объединяет функции физического и канального уровней модели OSI.

**Функции**:
- Физическое соединение с сетью
- Управление доступом к среде
- Работа с MAC-адресами

**Примеры протоколов**:
- Ethernet
- Wi-Fi (IEEE 802.11)
- PPP

#### 2. Интернет-уровень (Internet Layer)

Соответствует сетевому уровню модели OSI.

**Функции**:
- Маршрутизация пакетов
- Логическая адресация (IP)
- Управление сетевыми ошибками

**Примеры протоколов**:
- IP (IPv4, IPv6)
- ICMP (ping, traceroute)
- IGMP (многоадресная рассылка)

#### 3. Транспортный уровень (Transport Layer)

Соответствует транспортному уровню модели OSI.

**Функции**:
- Надёжная доставка данных (TCP)
- Быстрая ненадёжная доставка (UDP)
- Управление портами
- Сегментация и сборка данных
- Идентификация приложений

**Примеры протоколов**:
- TCP (Transmission Control Protocol)
- UDP (User Datagram Protocol)
- SCTP

#### 4. Прикладной уровень (Application Layer)

Объединяет функции сеансового, представления и прикладного уровней модели OSI.

**Функции**:
- Взаимодействие с приложениями
- Преобразование формата данных
- Управление пользовательскими сессиями

**Примеры протоколов**:
- HTTP (веб)
- SMTP, POP3, IMAP (почта)
- FTP (файлы)
- DNS (имена доменов)
- SSH (защищённый доступ)
- Telnet (удалённый доступ)

### Сравнение OSI и TCP/IP

| Аспект | OSI | TCP/IP |
|--------|-----|--------|
| Количество уровней | 7 | 4 |
| Тип модели | Концептуальная | Практическая |
| Разработка | Теоретическая | Эмпирическая (из реальной практики) |
| Использование | Обучение, стандартизация | Интернет, реальные сети |
| Детализация | Высокая | Менее детальная |

### Инкапсуляция в TCP/IP

Данные упаковываются на каждом уровне:

```
Приложение: Данные (например, HTML-страница)
    ↓
Транспортный уровень: Данные + заголовок TCP (порт, контрольная сумма)
    ↓
Интернет-уровень: Сегмент + заголовок IP (IP-адреса)
    ↓
Сетевой интерфейс: Пакет + заголовок Ethernet (MAC-адреса)
    ↓
Физический уровень: Кадр Ethernet отправляется в виде битов
```

На получающей стороне происходит де-инкапсуляция (обратный процесс).

## Архитектуры сетей

### Архитектура Ethernet (IEEE 802.3)

**Ethernet** — это наиболее распространённая архитектура локальных сетей.

**Характеристики**:
- **Физическая топология**: Изначально шина, но чаще звезда
- **Логическая топология**: Общей шины
- **Скорость**: От 10 Мбит/с до 100 Гбит/с и выше
- **Метод доступа**: CSMA/CD (Carrier Sense Multiple Access with Collision Detection)

**Метод CSMA/CD**:
1. Прослушивание канала перед передачей
2. Передача данных, если канал свободен
3. Обнаружение коллизий (столкновений сигналов)
4. Отправка сигнала задержки при коллизии
5. Повторная попытка передачи

**Стандарты**:
- 10BASE-T: 10 Мбит/с, витая пара, 100 м
- Fast Ethernet (100BASE-T): 100 Мбит/с
- Gigabit Ethernet (1000BASE-T): 1 Гбит/с
- 10 Gigabit Ethernet: 10 Гбит/с

**Пример**: Корпоративная локальная сеть с коммутатором Ethernet, к которому подключены компьютеры через витую пару.

### Архитектура Token Ring (IEEE 802.5)

**Token Ring** использует кольцевую топологию с методом передачи токена.

**Характеристики**:
- **Топология**: Кольцо
- **Скорость**: 4 или 16 Мбит/с
- **Метод доступа**: Передача маркера (token)

**Как работает**:
1. Специальный пакет (маркер) циркулирует по кольцу
2. Компьютер, имеющий маркер, может передавать данные
3. После передачи маркер передаётся следующему компьютеру

**Примечание**: Token Ring в основном устарела и редко используется.

### Архитектура FDDI (Fiber Distributed Data Interface)

**FDDI** — это расширение Token Ring для высокоскоростных сетей с использованием волоконной оптики.

**Характеристики**:
- **Топология**: Двойное кольцо
- **Скорость**: 100 Мбит/с
- **Среда**: Волоконная оптика
- **Надёжность**: Высокая (благодаря двойному кольцу)

**Преимущества**:
- Высокая надёжность
- Высокая скорость
- Большие расстояния

**Недостатки**:
- Дороговизна
- Сложность установки

## Сетевые протоколы

### Протоколы уровня приложений

#### HTTP (HyperText Transfer Protocol)

**Назначение**: Передача веб-страниц.

**Характеристики**:
- Протокол без сохранения состояния (stateless)
- Использует порт 80 (или 8080, 8000 и т.д.)
- HTTPS (с портом 443) — защищённая версия с шифрованием SSL/TLS

**Пример запроса**:
```
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
```

**Пример ответа**:
```
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>...</html>
```

#### SMTP (Simple Mail Transfer Protocol)

**Назначение**: Отправка электронной почты.

**Характеристики**:
- Использует порт 25 (или 587 с аутентификацией)
- Работает с SMTP-сервером

**Пример**: Когда вы отправляете письмо из Outlook, используется SMTP.

#### POP3 (Post Office Protocol)

**Назначение**: Получение электронной почты.

**Характеристики**:
- Использует порт 110
- Загружает письма на локальный компьютер

#### FTP (File Transfer Protocol)

**Назначение**: Передача файлов между компьютерами.

**Характеристики**:
- Использует порты 20-21
- Требует аутентификации
- Менее безопасен, чем SFTP

#### DNS (Domain Name System)

**Назначение**: Преобразование доменных имён в IP-адреса.

**Характеристики**:
- Использует порт 53
- Работает с иерархической базой данных доменов
- Кэширует результаты для быстродействия

**Пример**: Когда вы вводите www.example.com, DNS преобразует это в IP-адрес.

#### SSH (Secure Shell)

**Назначение**: Защищённый удалённый доступ и выполнение команд.

**Характеристики**:
- Использует порт 22
- Использует криптографию для защиты
- Замена небезопасного Telnet

### Протоколы транспортного уровня

#### TCP (Transmission Control Protocol)

**Характеристики**:
- Надёжная доставка (гарантирует доставку всех пакетов)
- Установление соединения (three-way handshake)
- Контроль потока (flow control)
- Управление перегруженностью (congestion control)
- Упорядоченная доставка

**Three-way handshake**:
```
Клиент: SYN (запрос соединения)
          →
Сервер: SYN-ACK (согласие)
          ←
Клиент: ACK (подтверждение)
          →
```

**Использование**: 
- Веб-браузеры (HTTP)
- Электронная почта
- Передача файлов
- Удалённый доступ

#### UDP (User Datagram Protocol)

**Характеристики**:
- Быстрая ненадёжная доставка
- Нет установления соединения
- Нет гарантии доставки
- Низкие задержки

**Использование**:
- Онлайн-игры
- Потоковое видео/аудио
- VoIP (голос через IP)
- DNS запросы

### Протоколы сетевого уровня

#### IP (Internet Protocol)

**Характеристики**:
- Логическая адресация (IP-адреса)
- Маршрутизация пакетов
- IPv4: 32-битные адреса (например, 192.168.1.1)
- IPv6: 128-битные адреса (например, 2001:0db8:85a3::8a2e:0370:7334)

#### ICMP (Internet Control Message Protocol)

**Использование**:
- ping (проверка доступности хоста)
- traceroute (отслеживание маршрута)
- Сообщения об ошибках

**Пример**: 
```
ping www.example.com
```

## Практические примеры

### Пример 1: Отправка электронного письма

**Сценарий**: Пользователь отправляет письмо с помощью Outlook.

**Процесс**:

1. **Прикладной уровень**: Пользователь компонует письмо в Outlook
   - Протокол: SMTP
   - Действие: Подготовка письма для отправки

2. **Транспортный уровень**: Outlook устанавливает соединение SMTP
   - Протокол: TCP
   - Действие: Установление надёжного соединения с SMTP-сервером (порт 587)

3. **Сетевой уровень**: Маршрутизация пакета на SMTP-сервер
   - Протокол: IP
   - Действие: Определение маршрута через маршрутизаторы

4. **Канальный уровень**: Доставка пакета в локальной сети
   - Протокол: Ethernet
   - Действие: Использование MAC-адреса маршрутизатора

5. **Физический уровень**: Передача электрических сигналов
   - Среда: Кабель Ethernet (витая пара)

### Пример 2: Открытие веб-сайта

**Сценарий**: Пользователь открывает www.example.com в браузере.

**Процесс по уровням**:

**Этап 1: Разрешение имени (DNS)**

```
Браузер → Локальный DNS-клиент → Рекурсивный резолвер → Корневой сервер DNS
                                    → Сервер .com
                                    → Авторитетный сервер example.com
                                    ← Возвращение IP-адреса 93.184.216.34
```

**Этап 2: Установление соединения (TCP)**

```
Браузер: SYN → Веб-сервер (порт 80)
Веб-сервер: SYN-ACK ←
Браузер: ACK →
Соединение установлено
```

**Этап 3: Отправка запроса (HTTP)**

```
Клиент → Веб-сервер:
GET / HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
```

**Этап 4: Получение ответа**

```
Веб-сервер → Клиент:
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>...</html>
```

**Уровневый анализ**:

| Уровень | Действие |
|---------|----------|
| Приложение (7) | Браузер формирует HTTP-запрос |
| Представление (6) | Кодирование данных (если необходимо) |
| Сеанс (5) | Управление соединением браузера |
| Транспорт (4) | TCP устанавливает соединение и передаёт HTTP-запрос |
| Сеть (3) | IP маршрутизирует пакет к веб-серверу |
| Канальный (2) | Ethernet доставляет кадр в локальной сети |
| Физический (1) | Электрические сигналы по кабелю |

### Пример 3: Локальная сеть с несколькими компьютерами

**Топология сети**:

```
                    [Коммутатор Ethernet]
                    /         |         \
                   /          |          \
              [ПК1]       [ПК2]       [ПК3]
            192.168.1.2  192.168.1.3  192.168.1.4
            MAC: AA      MAC: BB      MAC: CC
```

**Сценарий**: ПК1 отправляет файл на ПК2 через сетевую папку.

**Процесс**:

1. **Сетевой уровень**: ПК1 определяет, что ПК2 находится в той же подсети
   - IP-адрес ПК2: 192.168.1.3
   - Маска подсети: 255.255.255.0
   - Заключение: нет необходимости в маршрутизаторе

2. **Канальный уровень**: ПК1 отправляет ARP-запрос
   - Запрос: "Какой MAC-адрес у хоста с IP 192.168.1.3?"
   - Ответ от ПК2: "Мой MAC-адрес: BB"

3. **Передача данных**:
   - ПК1 инкапсулирует файл в кадр Ethernet
   - Заголовок Ethernet содержит:
     - MAC-адрес источника: AA
     - MAC-адрес получателя: BB
   - Коммутатор пересылает кадр на порт ПК2

4. **Получение**:
   - ПК2 получает кадр
   - Проверяет MAC-адрес (совпадает с его)
   - Распаковывает данные из кадра
   - Сохраняет файл

## Заключение

Понимание основных принципов построения компьютерных сетей включает знание:

1. **Топологий** — физического и логического расположения устройств

Эти концепции являются основой для построения, управления и устранения неполадок в компьютерных сетях любого размера — от малых локальных сетей до глобального интернета.
