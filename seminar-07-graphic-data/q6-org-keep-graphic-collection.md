
**Минусы:** Один файл может относиться к нескольким темам

## 4. Работа с метаданными

### Типы метаданных:

**EXIF** (автоматические):
- Техническая информация с камеры
- Дата, время, параметры съемки
- GPS-координаты

**IPTC** (добавляемые вручную):
- Ключевые слова (теги)
- Заголовок и описание
- Информация об авторе

### Правила добавления тегов:
- Отвечай на вопросы: Кто? Что? Где? Когда?
- Используй конкретные термины
- Добавляй эмоции/концепции

**Пример тегов для фотографии:**
`университет, защита_диплома, 2024, [имя], радость, аудитория_310`

## 5. Программы для работы

### Для начального уровня:
- **Adobe Bridge** — мощный браузер файлов
- **FastStone Image Viewer** — бесплатный просмотрщик
- **XnView MP** — кроссплатформенное решение

### Для профессионального роста:
- **Adobe Lightroom Classic** — золотой стандарт
- **Capture One** — альтернатива для коммерческой фотографии

**Важное правило:** Не перемещай файлы через проводник — используй интерфейс программы!

## 6. Стратегия хранения: Правило 3-2-1

### Что это значит:
- **3 копии** данных
- **2 разных** носителя
- **1 копия** удаленно

### Пример для студента:

| Копия | Носитель | Местоположение |
|-------|----------|----------------|
| Рабочая | SSD ноутбука | С тобой |
| Резервная 1 | Внешний HDD | Дома/в общежитии |
| Резервная 2 | Облако (Яндекс.Диск и др.) | Удаленно |

## 7. Практические рекомендации

### Именование файлов:
**Плохо:** `IMG_1234.JPG`, `Финальный_правка_NEW.psd`

**Хорошо:** `20240521_Курсовой_Вебсайт_Лендинг_v3.psd`

Формат: `ГГГГММДД_Проект_Описание_Версия.расширение`

### Версионность:
- Сохраняй ключевые версии
- Не перезаписывай файлы
- Используй нумерацию: `v1`, `v2`, `FINAL`

### Регулярность:
- 15 минут в неделю на организацию
- Сразу добавляй теги к новым файлам
- Своевременно делай бэкапы
