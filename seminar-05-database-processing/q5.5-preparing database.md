# **ОСНОВЫ ПОДГОТОВКИ ТАБЛИЧНЫХ ДАННЫХ ДЛЯ ПОСЛЕДУЮЩЕГО АНАЛИЗА**

Подготовка табличных данных — это важный этап аналитического цикла, от которого напрямую зависят точность, достоверность и скорость последующего анализа. Неподготовленные данные, содержащие ошибки, пропуски и несогласованности, могут привести к принятию ошибочных решений.

## **1. Цели и важность подготовки данных**

Основная цель подготовки данных — преобразовать исходную информацию в пригодный для анализа вид, обеспечив ее **точность, полноту и непротиворечивость**.

**Ключевые задачи этого этапа:**

- **Повышение качества анализа:** качественные данные — основа достоверных выводов.
- **Сокращение времени на анализ:** автоматизация очистки данных экономит часы ручной работы.
- **Обеспечение совместимости:** приведение данных из разных источников к единому стандарту позволяет их корректно сравнивать и агрегировать.
- **Автоматизация отчетности:** создание *«чистых»* данных позволяет строить динамические отчеты и дашборды, которые обновляются автоматически.

## **2. Ключевые принципы *«чистых»* данных**

1. **Принцип «Одна таблица — один тип объекта»**  
   Каждая таблица должна описывать один класс объектов. Связь между ними устанавливается через уникальные идентификаторы (ID).

2. **Принцип «Одна строка — один наблюдение»**  
   Каждая строка должна содержать данные об одном уникальном объекте или событии. Не допускается объединение нескольких записей в одну строку.

3. **Принцип «Один столбец — один атрибут»**  
   Каждый столбец должен содержать данные только одного типа и описывать один параметр объекта.

4. **Принцип «Первой строкой — заголовки»**  
   Первая строка таблицы должна содержать краткие, уникальные и понятные названия столбцов.

5. **Принцип «Без объединенных ячеек»**  
   Объединенные ячейки нарушают структуру данных и создают серьезные проблемы для сортировки, фильтрации и использования сводных таблиц. Их использование в теле таблицы недопустимо.

6. **Принцип «Без пустых строк и столбцов»**  
   Пустые строки и столбцы, часто используемые для визуального разделения, мешают корректной работе инструментов анализа. Для разделения данных следует использовать отдельные листы или таблицы.

## **3. Основные этапы подготовки данных**

### **Этап 1: Сбор и первичный аудит**

- **Сбор данных:** консолидация информации из различных источников (*CSV-файлы, Google Forms, базы данных*).
- **Первичный осмотр:** оценка общего объема данных, просмотр структуры, выявление очевидных аномалий и пропусков.

### **Этап 2: Очистка и стандартизация**

- **Устранение дубликатов:** поиск и удаление полностью или частично совпадающих строк.
- **Обработка пропусков (пустых ячеек):**
  - *Анализ причины:* пропуск случайный или закономерный?
  - *Стратегии устранения:*
    - Удаление строки (если пропусков мало и они случайны)
    - Заполнение константой (например, «0» или «Не указано»)
    - Заполнение статистическими показателями (средним, медианой — для числовых данных)
    - Заполнение наиболее частым значением (для категориальных данных)
- **Исправление ошибок и опечаток:**
  - Поиск орфографических ошибок в текстовых данных
  - Проверка числовых данных на наличие выбросов (слишком больших или малых значений)
  - Стандартизация написания: приведение к одному регистру, унификация форматов написания
- **Приведение типов данных:** убедиться, что числа сохранены как числа, даты — как даты, а текст — как текст. Частая ошибка — числа с дополнительными пробелами или апострофами, которые Excel воспринимает как текст.

### **Этап 3: Преобразование и обогащение**

- **Разделение столбцов:** если в одном столбце объединены несколько атрибутов, их необходимо разделить на отдельные столбца.
- **Создание производных признаков:** на основе имеющихся данных можно создать новые, более информативные столбцы.
- **Нормализация данных:** Приведение данных к определенным диапазонам.

### **Этап 4: Верификация и сохранение**

- **Контрольная проверка:** применение фильтров, проверка итоговых сумм, выборочная сверка данных после всех преобразований.
- **Сохранение результата:** сохранение очищенных данных в новом файле или на новом листе, чтобы всегда можно было вернуться к исходному набору.

## **4. Инструменты и методы для подготовки данных в Excel/Google Таблицах**

- **Удаление дубликатов:** данные → удалить дубликаты
- **Поиск и замена (Ctrl+H):** для массового исправления ошибок и стандартизации
- **Функции для очистки текста:**
  - `СЖПРОБЕЛЫ()` — удаляет лишние пробелы
  - `ПРОПИСН()`, `СТРОЧН()`, `ПРОПНАЧ()` — приводят текст к одному регистру
  - `ЛЕВСИМВ()`, `ПРАВСИМВ()`, `ПСТР()` — для извлечения частей текста
- **Функции для работы с датами:** `ДЕНЬ()`, `МЕСЯЦ()`, `ГОД()` — для создания новых признаков
- **Логические функции:** `ЕСЛИ()`, `ЕСЛИОШИБКА()` — для обработки ошибок и условных преобразований
- **«Текст по столбцам»:** инструмент для разделения данных в одном столбце по разделителю (запятая, точка с запятой, пробел) или фиксированной ширине
- **Сводные таблицы:** отличный инструмент для проверки данных и выявления ошибок перед глубоким анализом

Подготовка табличных данных — это фундаментальная аналитическая дисциплина. Следование принципам «чистых» данных и методичное выполнение этапов их очистки и преобразования позволяют заложить прочный фундамент для любого последующего анализа.
